# 经营者流程紧凑审查（改版前）

审查范围：经营驾驶舱 → 经营结果与证据 → 我的广告 Agent → 待审批事项。视口 1440 × 1024；证据来自本轮浏览器截图。

## 步骤与健康度

| 步骤 | 截图 | 健康度 | 证据结论 |
|---|---|---|---|
| 1. 经营驾驶舱 | `audit-before/01-dashboard.png` | 需重构 | 首屏是单店、单 Agent 和 ACoS 状态；缺少全部店铺营业额、利润金额、利润率红线、订单量与趋势。技术责任链占据老板首屏。 |
| 2. 经营结果 | `audit-before/02-results.png` | 严重偏技术 | 默认只显示 Package、Skill、Tool、Observation/Measure；无法先判断经营目标、护栏和趋势。 |
| 3. 我的广告 Agent | `audit-before/03-agent.png` | 不完整 | 只有占位说明，无法理解 Agent 汇报、协作、经营责任和安全状态。 |
| 4. 待审批事项 | `audit-before/04-approvals.png` | 可用但割裂 | 审批本身可操作，但与异常、需确认、恢复、通知分散在多个一级入口。 |

## 优点

- 当前身份、Workspace、权限与假数据环境持续可见。
- 高风险按钮有文字标签，不只依赖颜色。
- 审批页保留方案比较、影响范围和回滚语义。

## UX 风险与改动

1. P1：老板首屏缺少聚合经营结果与趋势。改为全部店铺驾驶舱，并增加店铺、站点、时间、品类、商品、Agent 筛选。
2. P1：利润率容易被理解为主目标。改为“利润金额主目标 + 利润率红线护栏”，营收、订单、现金流作为辅助关键结果。
3. P1：经营结果默认暴露技术对象。改为四层下钻：目标/OKR → 关键结果与护栏 → 趋势与贡献 → Agent 行动/证据。
4. P1：Agent 页面为空。增加产品级汇报关系、协作关系、经营表现和安全控制。
5. P1：审批、异常和恢复入口割裂。统一为“收件箱”，驾驶舱保留摘要。
6. P2：改变记录使用系统语言。改为“经营动态 / 决策记录”，明确发生、原因、结果和负责人。
7. P2：紧急停止占据常用一级导航。移入 Agent 安全控制；风险状态下从驾驶舱与收件箱快捷到达。

## Accessibility 风险

- 改版前信息卡片语义较清晰，但趋势缺少图文双重表达。
- 改版后趋势必须同时提供数值、同比/环比文字和视觉条形，不依赖颜色。
- 截图不能证明完整键盘顺序、屏幕阅读器公告或缩放兼容；这些需浏览器交互验证。

影响分类：上述均为 `no-change` 产品原型优化。Agent 汇报/协作仅作为可理解的产品级关系展示，不声称新增核心对象或后端契约；建议记录 `Architecture review` finding，供技术负责人确认未来落模方式。
