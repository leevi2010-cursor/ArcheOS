# Tolaria 内容归类指南

对每篇现有笔记依次判断：

1. 它记录的是一次具体发生的事情吗？
   - 是：Issue / Case / Meeting / Memory。
2. 它描述稳定概念或背景知识吗？
   - 是：Knowledge / Tutorial。
3. 它是在约束选择、风险或行为吗？
   - 是：Protocol。
4. 它是在描述有顺序、有状态的过程吗？
   - 是：Workflow。
5. 它描述系统可验证地“会做什么”吗？
   - 是：Capability。
6. 它提供可运行的具体实现吗？
   - 是：Skill。
7. 它说明怎样判断好坏吗？
   - 是：Evaluation。
8. 它记录为何选择某个架构方案吗？
   - 是：Decision / ADR。

## 提升规则

Tolaria 中的草稿不会自动成为 ArcheOS 核心规范。

只有满足以下条件才提升：

- 已在真实业务中出现；
- 具有复用价值；
- 边界明确；
- 有反例或适用范围；
- 可被评价；
- 已通过架构审查。

## 降级规则

出现以下情况应保留在领域层或笔记层：

- 只服务单个客户；
- 尚未验证；
- 依赖临时上下文；
- 只是个人想法；
- 无法给出稳定输入输出或适用范围。
