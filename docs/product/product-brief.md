# 元枢 ArcheOS Product Brief

## 产品定位

元枢 ArcheOS 是一个帮助平台管理者孵化、部署和持续改进业务 Agent，并帮助经营者使用 Agent 经营业务的内部平台。v0.1 首先服务于 Leo 自己经营的亚马逊店铺，目标是让 Agent 在经营者设定的方向和授权范围内持续创造可验证的业务价值。

## 目标用户

- 直接用户和最终验收人：Leo。
- 平台管理者身份：管理领域知识与可复用能力、平台默认规则、Agent 创建和部署。
- 店铺经营者身份：选择经营导向、风险偏好和店铺级限制，使用 Agent 经营亚马逊店铺。
- 业务责任：Leo 作为店铺拥有者和经营者，对整体经营结果负责。
- v0.1 不存在其他内部委托人。

## 用户问题

当前亚马逊店铺运营依赖经营者跨系统获取数据、发现问题、判断、执行和复盘。工作难以连续自动运行，经验也难以沉淀为可复制、可独立部署并能持续改进的业务 Agent。

## 产品承诺

元枢帮助 Leo 孵化一个能够独立承担亚马逊广告运营职责的 Agent。这个 Agent 可以持续运营广告、观察经营结果、发现自身问题，并在安全边界内创建或修改自己的工作方法，使经营效果不断改善。

## v0.1 核心使用场景

1. Leo 以平台管理者身份管理亚马逊广告领域资产、默认规则，并孵化和部署广告运营 Agent。
2. Leo 以店铺经营者身份选择利润、现金流、销量、订单或清库存等经营导向，以及风险偏好、店铺级限制和必须人工确认的事项；也可以直接采用系统默认规则。
3. Agent 自动完成广告数据获取、问题诊断、决策、授权范围内的执行和结果复盘。
4. Agent 根据实际经营反馈发现能力缺口，提出并完成可验证、可撤销的自我改进；超出授权或可能显著影响经营结果的改变交由 Leo 审批。
5. 同一个 Agent 可以被复制到独立运行环境，通过少量必要配置开始独立运营。

## v0.1 范围

- 支持一个亚马逊店铺的广告自动化运营。
- 支持 Agent 从创建、配置、运行、观察到持续改进的完整生命周期。
- 支持 Agent 在授权边界内创建或修改自己的工作方法，并验证改进效果。
- 支持将 Agent 部署到相互隔离的容器中独立运行。
- 支持经营偏好引导、默认规则以及店铺凭证与广告访问凭证的配置帮助。
- 支持部署所需的少量关键配置，例如大模型 API Key、店铺与广告凭证和授权范围。
- 为 Agent 后续复制、交付和售卖验证必要的独立性与可配置性。

## 非目标

- 在 v0.1 实现覆盖店铺全部职能的无人值守运营。
- 在 v0.1 建设公开 Agent 市场、支付结算或完整交易系统。
- 允许 Agent 绕过 Leo 的经营目标、授权边界或审批要求。
- 要求 v0.1 首次验收就部署到另一台实体电脑或客户服务器。
- 替代亚马逊、广告平台或其他业务系统。

## v0.1 成功信号

1. **广告独立运营**：Agent 能持续完成广告数据获取、诊断、决策、执行和复盘，并根据经营者选择的优先导向及安全约束运营。
2. **Agent 自动孵化**：平台能够创建一个具备明确广告运营职责、运行配置和安全边界的可运行 Agent。
3. **Agent 持续进化**：Agent 能依据经营结果和运行反馈，自动创建或修改自己的工作方法；每次改变都能验证效果，失败时能够撤销，关键改变需经 Leo 审批。
4. **低成本独立部署**：同一个 Agent 能部署到另一个相互隔离的容器中，在完成少量关键配置后独立运行，不依赖原容器的临时状态。
5. **可复制基础**：部署过程、必要配置和运行边界足够清晰，使未来迁移到另一台电脑或用户服务器不需要重新设计 Agent。

## 后续产品方向

- 把验证成熟的 Agent 复制到其他店铺或经营场景。
- 将 Agent 部署到用户自己的电脑或服务器。
- 建立 Agent 的交付、授权、升级和售卖能力。
